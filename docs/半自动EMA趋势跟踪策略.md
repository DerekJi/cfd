代码每5分钟检查一次。下面以多头为例：



* 如果某品种当天没有指示分析，则不做任何处理。
* 如果是A类趋势行情，那么当价格回落并再次启动时，沿趋势方向自动开仓。具体来说，

&nbsp;   - 设定5分钟级别EMA均线组，比如\[ 20, 60, 80 ]，每个均线都是可能的关键位的动态参考指标，所以，它们每一个都是等价的。EMA20相比EMA60或EMA80，没有任何特殊性。

&nbsp;       - 均线组中，同时增加一条特别的H1均线：人工指示中的H1均线（需要使用OANDA API获取H1数据来计算）

&nbsp;   - 当价格回落到EMA均线组的任意一个（也可能极为接近但未触碰，比如：Price < EMA + 0.2 \* ATR）时，认为价格已回落到关键位置，可以设定状态为ON\_KEY\_POS\_{EMA\_NAME}，比如ON\_KEY\_POS\_20；

    - 当价格在回踩EMA位置后，再次阳线突破——收盘高于EMA且高于前3根K线的高点（close > EMA \&\& close > high(ago = -1, 3)），那么状态改为TRIGGERED；

&nbsp;   - 如果价格回落到EMA后并未回升，而且到达下一个时，状态修改为下一个，比如ON\_KEY\_POS\_60。此时，认为之前的EMA（比如EMA20）并不是关键位，当前的才是。

&nbsp;   - 状态为TRIGGERED时，立即开仓，并把止损点设置为突破前的低点稍低的位置，并设置状态为POSITION\_OPEN。这里，TRIGGERED和POSITION\_OPEN分为两个状态，似乎没什么必要性，但后面我可能会在中间插入一个人工或AI确认的环节。

&nbsp;       - 如果价格持续跌破最低一个EMA（比如n个ATR），那么，状态归零，当天不再开仓。因为，这样等于宣告所有的EMA均线在今天来说都不是关键位；

&nbsp;       - 如果价格跌破H1的EMA均线，那么把状态设置为INVALID，当天不再开仓。因为，这样等于宣告H1级别的这个EMA也不是关键位置。

* 如果是B类震荡行情，那么当价格出现突破时（只是简单突破，不需要考虑有效性），给我发个消息，我自己来判断突破的有效性等情况。消息发送后，30分钟内不再重复发送。
* FSM 状态机的健壮性（纠错机制）

&nbsp;      应避免“僵尸状态”，比如状态卡在 ON\_KEY\_POS 但价格早已飞走。

&nbsp;      - 时间戳校验（TTL）： 每个状态增加一个 last\_seen\_timestamp。比如，如果一个品种处于 ON\_KEY\_POS 超过 4 小时仍未 TRIGGERED，系统应强制将其重置为 IDLE。

&nbsp;      - 空间偏移重置： 如果价格反弹过快，直接拉升超过了前高，但未触发“回踩开仓”逻辑，说明回踩形态已消失。此时也应重置状态。

&nbsp;      - Oanda 订单同步： \* 每 5 分钟轮询开始时，先查一遍 Oanda 的 Trade List。

&nbsp;           + 如果数据库里是 POSITION\_OPEN 但 Oanda 里没单子，说明被手动平仓或扫损了，立即将 Table Storage 状态同步为 IDLE。           

&nbsp;           + 反之，如果 Oanda 有单子但数据库没记录，发 Telegram 报警。
