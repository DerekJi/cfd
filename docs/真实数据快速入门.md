# 真实数据使用快速入门

## 🚀 三步开始使用真实数据

### Step 1: 下载数据

选择以下任一方式：

**方式A：使用OANDA API（推荐）**
```bash
cd backtest
python download_data.py
# 选择选项 1，输入API密钥
```

**方式B：从HistData.com手动下载**
1. 访问 https://www.histdata.com/download-free-forex-data/
2. 下载 EUR/USD M1 数据
3. 运行转换工具：
   ```bash
   python convert_histdata.py
   ```

**方式C：使用MetaTrader 5**
```bash
python download_data.py
# 选择选项 2
```

---

### Step 2: 验证数据

```bash
python test_data.py
```

这会检查：
- ✅ 文件是否存在
- ✅ 格式是否正确
- ✅ 数据质量如何
- ✅ 时间范围是否充足

---

### Step 3: 运行回测

修改 `run_backtest.py` 最后一行：

```python
# 使用真实数据
run_backtest(use_sample_data=False)
```

然后运行：

```bash
python run_backtest.py
```

---

## 📊 数据要求

### 最低要求
- **时间范围**: 至少3个月
- **K线数量**: 至少25,000根
- **格式**: CSV，包含 datetime,open,high,low,close,volume

### 推荐配置
- **时间范围**: 6-12个月
- **K线数量**: 50,000-100,000根
- **包含**: 不同市场环境（震荡、趋势、高波动）

## ⚠️ 常见问题

### Q: 数据文件必须叫这个名字吗？
A: 不必须，可以在 `config.py` 中修改 `DATA_CONFIG['data_path']`

### Q: 可以使用其他货币对吗？
A: 可以，修改配置即可。策略逻辑是通用的。

### Q: 数据缺失怎么办？
A: 外汇市场周末休市是正常的。只要覆盖率>70%即可。

### Q: 回测很慢怎么办？
A: 
- 减少数据量（只用6个月数据）
- 调整参数（增大lookback_period）
- 禁用debug模式

### Q: 从哪里能看到真实信号？
A: 在终端输出中会显示状态转换和交易记录
