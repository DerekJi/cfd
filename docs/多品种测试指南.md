# 多品种回测测试指南

## 📋 测试目的

验证趋势过滤策略在不同交易品种上的表现，以确定：
1. **策略稳健性**：在不同品种上表现是否均衡
2. **参数优化**：是否需要为不同品种设置不同参数

## 🎯 测试品种

当前支持的品种：

| 品种代码 | 名称 | 类型 | Pip值 |
|---------|------|------|-------|
| GBP_USD | 英镑/美元 | 外汇 | 0.0001 |
| EUR_USD | 欧元/美元 | 外汇 | 0.0001 |
| XAU_USD | 黄金 | 贵金属 | 0.01 |
| XAG_USD | 白银 | 贵金属 | 0.001 |
| USD_JPY | 美元/日元 | 外汇 | 0.01 |
| AUD_USD | 澳元/美元 | 外汇 | 0.0001 |

## 🚀 使用步骤

### 步骤1：下载多品种数据

```bash
# 下载所有品种数据（需要OANDA API）
cd backtest
python download_multi_symbols.py
```

**注意事项：**
- 需要在 `oanda_config.py` 中配置OANDA API密钥
- 下载时间较长（每个品种约1-2分钟）
- 如果某个品种下载失败，可以单独重试

### 步骤2：运行多品种测试

```bash
# 运行所有品种的参数测试
python multi_symbol_test.py
```

测试会自动：
1. 遍历所有已下载的品种
2. 测试不同的ADX阈值（20, 25, 30）
3. 生成汇总报告

### 步骤3：分析结果

测试完成后会显示：

**1. 各品种最佳参数**
```
品种       ADX阈值    收益率    胜率      盈亏比    回撤      交易数
----------------------------------------------------------------------
GBPUSD     25         +0.38%    24.3%     3.18      3.28%     622
EURUSD     20         +1.52%    28.5%     2.85      2.95%     745
...
```

**2. 稳健性分析**
- 平均收益
- 收益标准差
- 收益范围
- 稳健性评估

## 📊 结果解读

### 场景A：表现均衡 ✅

**特征：**
- 各品种收益率接近（标准差<2%）
- 盈亏比都在2.5-3.5之间
- 最佳参数相同或接近

**结论：**
- 策略稳健，可用统一参数
- 建议使用中等ADX阈值（如25）

**示例：**
```
GBPUSD: +0.38%, 盈亏比3.18, ADX=25
EURUSD: +0.52%, 盈亏比3.05, ADX=25
AUDUSD: +0.41%, 盈亏比3.22, ADX=25
```

### 场景B：需要品种特定参数 ⚠️

**特征：**
- 各品种最佳参数差异大
- 使用统一参数某些品种表现差
- 但使用最佳参数都能盈利

**结论：**
- 策略核心逻辑有效
- 需要为不同品种优化参数

**示例：**
```
GBPUSD: +0.38%, 最佳ADX=25
XAUUSD: +2.15%, 最佳ADX=30  （黄金波动大，需要更严格过滤）
USDJPY: +0.95%, 最佳ADX=20  （日元趋势更明显）
```

### 场景C：策略不稳健 ❌

**特征：**
- 某些品种亏损严重
- 更换参数也无法改善
- 盈亏比差异极大

**结论：**
- 策略可能只适合特定品种
- 需要重新设计策略逻辑

## 🔧 参数调优建议

### ADX阈值影响

| ADX阈值 | 效果 | 适用场景 |
|--------|------|---------|
| 15-20 | 交易频率高，趋势要求低 | 趋势较明显的品种 |
| 20-25 | 平衡，中等过滤强度 | 通用设置 |
| 25-30 | 交易频率低，只抓强趋势 | 震荡频繁的品种 |
| >30 | 极少交易，极强趋势 | 不推荐 |

### 不同品种特性

**外汇主要货币对（EURUSD, GBPUSD, AUDUSD）：**
- 流动性好，点差小
- 趋势相对温和
- 建议ADX: 20-25

**贵金属（XAUUSD, XAGUSD）：**
- 波动性大，趋势强烈
- 假突破相对少
- 建议ADX: 25-30

**日元货币对（USDJPY）：**
- 趋势持续性好
- 回调较浅
- 建议ADX: 20，可能需要调整回踩判断

## 💡 高级使用

### 自定义测试参数

编辑 `multi_symbol_test.py` 中的 `PARAM_GRID`：

```python
PARAM_GRID = {
    'adx_threshold': [15, 20, 25, 30],  # 测试更多阈值
    'ema50_period': [40, 50, 60],       # 测试不同EMA周期
    'ema200_period': [180, 200, 220],
    'up_period': [15, 20, 25],          # 测试不同Donchian周期
}
```

### 添加新品种

在 `SYMBOLS` 字典中添加：

```python
SYMBOLS = {
    # ... 现有品种
    'EUR_GBP': {
        'file': './data/EUR_GBP_M5.csv',
        'pip_value': 0.0001,
        'description': 'EUR/GBP - 欧元/英镑'
    },
}
```

## ⚠️ 注意事项

1. **数据质量**：确保所有品种的数据时间范围一致
2. **计算时间**：测试多个品种+多个参数会花费较长时间
3. **过拟合风险**：不要过度优化参数，保留验证集
4. **实盘差异**：回测结果仅供参考，实盘需考虑滑点、点差等

## 🎓 下一步

根据测试结果：

**如果表现均衡：**
1. 使用统一的最佳参数
2. 可以开始实盘测试
3. 监控各品种表现差异

**如果需要优化：**
1. 为不同品种保存最佳参数配置
2. 创建品种特定的策略实例
3. 可能需要调整核心逻辑（如回踩判断）

**如果不稳健：**
1. 分析原因（品种特性、参数敏感度）
2. 考虑只用于表现好的品种
3. 或者重新设计策略逻辑
