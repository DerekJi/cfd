## 阶段 A — Telegram Bot 命令测试

**前置条件**
- Azure Functions 已部署，`telegram_webhook` HTTP Trigger 可访问
- 已在 BotFather 设置 Webhook：`https://<func>.azurewebsites.net/api/telegram_webhook`（附 `?code=` 或 secret header）
- `CFD_DRY_RUN=true`（全程 dry run，不真实下单）

## 阶段 B — 策略端到端测试

**目标**：触发真实 H1/5M 扫描，验证信号产生 → Bot 通知 → 手动确认开仓 → 调整 SL → 平仓的完整链路。

### B-1 准备测试品种

```
/watch add EUR_USD
/watch add USD_JPY
/range clear EUR_USD        ← 清除区间，让信号更容易击穿
/set ema_h1 21 55 200
/set ema_5m 10 20 50
```

### B-2 H1 扫描（`semi_auto_h1_scan`）

**触发方式**（二选一）：
- 等待自然整点（:00:05）
- 在 Azure Portal → Functions → `semi_auto_h1_scan` → "Test/Run" 手动执行

**验收标准**：
1. Function 日志显示 `[semi_auto_h1_scan] done: triggered=X`
2. 若有穿越信号：Telegram 收到图表消息，含品种名、方向、建议 `/trend` 命令
3. 若无信号：日志 `skipped=[...]`，Telegram 无多余噪音消息

执行：
```
/trend long EUR_USD 21      ← 手动激活，模拟 H1 已发出信号
```

### B-3 5M 扫描（`semi_auto_5m_scan`）

**触发方式**：同上，手动 "Test/Run" 或等待 :00:10 / :05:10 ...

**验收标准**：
1. Function 日志显示 `[semi_auto_5m_scan] done`
2. 若检测到顺势 5M 信号：
   - Telegram 收到 5M 图表，含入场价/止损价/手数
   - 消息末尾含操作按钮提示：`/open EUR_USD` 或 `/cancel EUR_USD /dnd 30 EUR_USD`
3. `/status` 显示 PendingSignal 已保存

### B-4 手动确认开仓

```
/open EUR_USD
```

**验收标准（DRY_RUN=true）**：
- 回复"模拟开仓成功"，含订单方向/手数/SL
- `/order list` 回复列出该笔仓位（executor.place_order 在 dry run 下应返回 mock id）
- PendingSignal 已清除：再发 `/open EUR_USD` 回复"无信号"

### B-5 调整止损

```
/move EUR_USD 1.0850
```

**验收标准**：
- 回复"SL 已更新为 1.0850"
- Oanda Demo 账户（或 dry run 日志）可见 SL 变更

### B-6 平仓

```
/close all
```

**验收标准**：
- 回复"已平仓 X 笔"
- `/order list` 回复为空

### B-7 品种 DND 覆盖测试

```
/trend long EUR_USD 21
/dnd 15 EUR_USD
```

立即手动触发 `semi_auto_5m_scan`：
- 日志显示 EUR_USD 因 DND 跳过
- Telegram 不收到 EUR_USD 信号

## 阶段 C — 异常与边界

| # | 场景 | 验证方式 |
|---|------|---------|
| C1 | Oanda API Key 错误 | Function 日志出现 `[semi_auto_h1_scan] failed`，Telegram 收到 `🚨 H1扫描异常` |
| C2 | Webhook 未设置 `CFD_TELEGRAM_WEBHOOK_SECRET`，外部 POST 请求 | 返回 200 OK（无 secret 时不校验），不崩溃 |
| C3 | Webhook POST 设置了 secret 但 header 不匹配 | 返回 403 |
| C4 | Webhook POST body 非 JSON | 返回 400 |
| C5 | 非授权 chat_id 发送任意命令 | 无回复，Function 日志 `Unauthorized access from chat_id=...` |
| C6 | `/open` 后立刻再 `/open`（重复确认）| 回复"无待确认信号"，不重复下单 |
