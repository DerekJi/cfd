# 半自动EMA趋势跟踪策略开发计划

## 现有代码盘点

| 模块 | 文件 | 可复用度 |
|---|---|---|
| OANDA K线拉取（支持多周期） | [data/oanda_candles.py](../../data/oanda_candles.py) | ✅ 直接复用 |
| EMA / ATR 指标计算 | [core/indicators.py](../../core/indicators.py) | ✅ 直接复用 |
| 仓位计算（含风控） | [core/position_sizer.py](../../core/position_sizer.py) | ✅ 调整风险参数 |
| OANDA 下单执行器 | [execution/oanda_executor.py](../../execution/oanda_executor.py) | ✅ 直接复用 |
| Telegram 单向通知 | [notification/telegram.py](../../notification/telegram.py) | ⚠️ 需大幅扩展（命令接收） |

## 开发计划（共 7 个阶段）

### Phase 1 — 数据层扩展（0.5天）

**目标**：支持同时拉取 H1 + 5M 两个周期的 K 线

- [x] 在 `OandaDataProvider` 中支持批量获取多周期（H1 × 200、5M × 200）
- [x] 封装 `get_multi_timeframe()` 方法，返回 `{H1: df, M5: df}`
- [x] 单元测试：验证 H1 和 M5 数据格式一致

---

### Phase 2 — EMA 扫描器（0.5天）

**目标**：实现有效穿越检测逻辑

- [x] 新建 `strategies/ema_scanner.py`，SRP 原则
- [x] 实现 `check_ema_crossover(df, ema_periods, atr, direction)` —— 有效穿越（±0.5×ATR）
- [x] 实现 `check_3bar_breakout(df, direction)` —— 最近3根K线高/低点过滤
- [x] 实现 `calculate_stop_loss(df, direction, buffer_pips)` —— 10根K线止损位
- [x] 单元测试覆盖多/空两个方向（27 个用例，全部通过）

---

### Phase 3 — 状态管理层（1天）

**目标**：持久化所有半自动策略所需的状态

- [x] 新建 `strategies/semi_auto/state.py`，定义数据模型：
  - `WatchlistItem`：品种、震荡区间上下限
  - `TrendPoolItem`：品种、方向(long/short)、绑定H1均线周期、加入时间
  - `PendingSignal`：待确认信号（品种、方向、入场价、止损、仓位）
  - `DndTimeSlot`：全局免打扰时段（UTC HH:MM 起止）
- [x] 扩展 `StateStorage` 接口（新增 10 个方法），支持上述状态读写
- [x] 在 `AzureTableStorage` 和 `LocalJsonStorage` 中实现新接口
- [x] 注意 DRY：通用 upsert/load 逻辑已有，直接复用

---

### Phase 4 — 图表生成（0.5天）

**目标**：生成 K线图 + EMA叠加图，发送到 Telegram

- [x] 新建 `utils/chart_generator.py`
- [x] 依赖 `matplotlib`，生成 PNG 字节流（自动检测系统 CJK 字体，Catppuccin 暗色主题）
- [x] 支持参数：K线数据、均线列表（含数值和颜色）、当前价格/入场价/止损位标注、方向箭头
- [x] 封装为 `generate_ema_chart(df, ema_dict, title) -> bytes`
- [x] 扩展 `TelegramNotifier.send_photo(image_bytes, caption)` 方法
- [x] 验证测试：`tests/test_phase4_chart.py`（EUR_USD M5 · 100根K线 · EMA10/20/50）

---

### Phase 5 — Telegram Bot 命令交互（1天）

**目标**：实现双向命令交互（接收 & 解析用户命令）

- [ ] 新建 `notification/telegram_bot.py`，基于 `python-telegram-bot` 库
- [ ] 实现所有命令处理器（SRP：每条命令一个 handler 函数）：
  - `/watch add/remove/list`
  - `/range set/clear/list`
  - `/trend long/short/clear/list`
  - `/dnd 15/30`、`/dnd global add/remove/list/clear`
  - `/set ema_h1/ema_5m`
  - `/open`、`/cancel`
  - `/status`、`/close all`、`/move sl`
  - `/order list`
- [ ] Bot 启动作为独立进程/线程，命令回调更新状态后触发相应动作
- [ ] 安全校验：只响应授权 chat_id

---

### Phase 6 — 半自动引擎（1.5天）

**目标**：编排两级扫描主循环

- [ ] 新建 `live/semi_auto_engine.py`（不影响现有 `engine.py`）

**H1 扫描循环**（每小时触发）：
- [ ] 遍历 Watchlist，拉取 H1×200
- [ ] 检查震荡区间 → 区间内跳过
- [ ] 计算 EMA 有效穿越 → 触发时发图+通知（含标记趋势建议按钮）

**5M 扫描循环**（每5分钟触发）：
- [ ] 检查全局免打扰时段 → 时段内静默
- [ ] 遍历趋势激活池，拉取5M×200 + H1×200（取绑定均线）
- [ ] 检查单品种免打扰 → 免打扰内静默
- [ ] 检查顺势穿越（5M EMA任意一条 或 H1绑定均线）
- [ ] 通过3根K线高/低点过滤
- [ ] 计算止损位、仓位（风险≤0.8%）→ 风控验证
- [ ] 风控通过 → 推送信号+开仓确认按钮（附带15/30分钟免打扰选项）
- [ ] 保存 `PendingSignal` 到状态存储

**开仓执行**（由 `/open` 触发）：
- [ ] 从状态存储加载 `PendingSignal`
- [ ] 通过 OANDA Stream API 获取实时价格
- [ ] 调用 `OandaExecutor.place_order()` 执行
- [ ] 发送开仓确认通知，清除 `PendingSignal`

---

### Phase 7 — 部署与集成（0.5天）

**目标**：接入现有 Azure Function 框架

- [ ] 在 `live/azure_function/function_app.py` 中添加两个 Timer Trigger：
  - `semi_auto_h1_scan`（每小时触发）
  - `semi_auto_5m_scan`（每5分钟触发）
- [ ] 添加一个 HTTP Trigger：接收 Telegram Webhook 回调
- [ ] 更新 `requirements.txt`（新增 `python-telegram-bot`、`mplfinance`）
- [ ] 补充 `local.settings.template.json` 中新增的环境变量
- [ ] 端到端测试：本地 Azurite + Telegram Bot 联调
