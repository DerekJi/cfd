# 半自动EMA趋势跟踪策略开发计划

## 现有代码盘点

| 模块 | 文件 | 可复用度 |
|---|---|---|
| OANDA K线拉取（支持多周期） | [data/oanda_candles.py](../../data/oanda_candles.py) | ✅ 直接复用 |
| EMA / ATR 指标计算 | [core/indicators.py](../../core/indicators.py) | ✅ 直接复用 |
| 仓位计算（含风控） | [core/position_sizer.py](../../core/position_sizer.py) | ✅ 调整风险参数 |
| OANDA 下单执行器 | [execution/oanda_executor.py](../../execution/oanda_executor.py) | ✅ 直接复用 |
| Telegram 单向通知 | [notification/telegram.py](../../notification/telegram.py) | ⚠️ 需大幅扩展（命令接收） |
| Azure Table 状态存储 | [storage/azure_table.py](../../storage/azure_table.py) | ⚠️ 需扩展表结构 |
| 实盘引擎（全自动） | [live/engine.py](../../live/engine.py) | ❌ 架构不同，参考用 |
| Azure Function 部署框架 | [live/azure_function/](../../live/azure_function/) | ✅ 复用部署结构 |

**核心缺口**：现有 Telegram 只能**发送**，无法**接收命令**；无半自动工作流、无图表生成、无震荡区间/免打扰管理。

---

## 开发计划（共 7 个阶段）

### Phase 1 — 数据层扩展（0.5天）

**目标**：支持同时拉取 H1 + 5M 两个周期的 K 线

- [x] 在 `OandaDataProvider` 中支持批量获取多周期（H1 × 200、5M × 200）
- [x] 封装 `get_multi_timeframe()` 方法，返回 `{H1: df, M5: df}`
- [x] 单元测试：验证 H1 和 M5 数据格式一致

---

### Phase 2 — EMA 扫描器（0.5天）

**目标**：实现有效穿越检测逻辑

- [x] 新建 `strategies/ema_scanner.py`，SRP 原则
- [x] 实现 `check_ema_crossover(df, ema_periods, atr, direction)` —— 有效穿越（±0.5×ATR）
- [x] 实现 `check_3bar_breakout(df, direction)` —— 最近3根K线高/低点过滤
- [x] 实现 `calculate_stop_loss(df, direction, buffer_pips)` —— 10根K线止损位
- [x] 单元测试覆盖多/空两个方向（27 个用例，全部通过）

---

### Phase 3 — 状态管理层（1天）

**目标**：持久化所有半自动策略所需的状态

- [x] 新建 `strategies/semi_auto/state.py`，定义数据模型：
  - `WatchlistItem`：品种、震荡区间上下限
  - `TrendPoolItem`：品种、方向(long/short)、绑定H1均线周期、加入时间
  - `PendingSignal`：待确认信号（品种、方向、入场价、止损、仓位）
  - `DndTimeSlot`：全局免打扰时段（UTC HH:MM 起止）
- [x] 扩展 `StateStorage` 接口（新增 10 个方法），支持上述状态读写
- [x] 在 `AzureTableStorage` 和 `LocalJsonStorage` 中实现新接口
- [x] 注意 DRY：通用 upsert/load 逻辑已有，直接复用

---

### Phase 4 — 图表生成（0.5天）

**目标**：生成 K线图 + EMA叠加图，发送到 Telegram

- [x] 新建 `utils/chart_generator.py`
- [x] 依赖 `matplotlib`，生成 PNG 字节流（自动检测系统 CJK 字体，Catppuccin 暗色主题）
- [x] 支持参数：K线数据、均线列表（含数值和颜色）、当前价格/入场价/止损位标注、方向箭头
- [x] 封装为 `generate_ema_chart(df, ema_dict, title) -> bytes`
- [x] 扩展 `TelegramNotifier.send_photo(image_bytes, caption)` 方法
- [x] 验证测试：`tests/test_phase4_chart.py`（EUR_USD M5 · 100根K线 · EMA10/20/50）

---

### Phase 5 — Telegram Bot 命令交互（1天）

**目标**：实现双向命令交互（接收 & 解析用户命令）

- [x] 新建 `notification/telegram_bot.py`，基于 `python-telegram-bot>=20` 库
- [x] 实现所有命令处理器（SRP：每条命令一个 async handler 方法）：
  - `/watch add/remove/list`
  - `/range set/clear/list`
  - `/trend long/short/clear/list`
  - `/dnd 15/30 SYMBOL`、`/dnd global add/remove/list/clear HH:MM HH:MM`
  - `/set ema_h1/ema_5m P1 [P2 ...]`
  - `/open [SYMBOL]`、`/cancel [SYMBOL]`
  - `/status`、`/close all`、`/move SYMBOL NEW_SL`
  - `/order list`、`/help`
- [x] 支持两种运行模式：`run_polling()`（本地开发）和 `process_update_sync(dict)`（Azure Webhook）
- [x] 安全校验：所有 handler 通过 `_guard` 装饰器过滤非授权 chat_id
- [x] EMA 设置持久化到 storage（key `__sa_settings__`），引擎启动时自动读取

---

### Phase 6 — 半自动引擎（1.5天）

**目标**：编排两级扫描主循环

- [x] 新建 `live/semi_auto_engine.py`（不影响现有 `engine.py`）

**H1 扫描循环**（每小时触发）：
- [x] 遍历 Watchlist，拉取 H1×250
- [x] 检查震荡区间 → 区间内跳过
- [x] 计算 EMA 有效穿越（多/空双向）→ 触发时发图+通知（提示 /trend 命令激活）

**5M 扫描循环**（每5分钟触发）：
- [x] 检查全局免打扰时段 → 时段内静默
- [x] 遍历趋势激活池，拉取 M5×250 + H1×250（取绑定均线）
- [x] 检查单品种免打扰 → 免打扰内静默
- [x] 检查顺势穿越（5M EMA任意一条 或 H1绑定均线，任一满足即可）
- [x] 通过3根K线高/低点过滤
- [x] 计算止损位（10根K线极值 + 5pip缓冲）、仓位（风险≤0.8%）→ 风控验证
- [x] 风控通过 → 保存 `PendingSignal` + 推送图表通知（含操作指令说明）

**开仓执行**（由 `/open` 触发）：
- [x] 从状态存储加载 `PendingSignal`
- [x] 调用 `OandaExecutor.place_order()` 市价成交
- [x] 发送开仓确认通知，清除 `PendingSignal`
- [x] `execute_close_all()`、`execute_move_sl()`、`get_open_positions()` 供 Bot 调用

---

### Phase 7 — 部署与集成（0.5天）

**目标**：接入现有 Azure Function 框架

- [x] 在 `live/azure_function/function_app.py` 中添加两个 Timer Trigger：
  - `semi_auto_h1_scan`（每小时触发，CRON: `5 0 * * * *`）
  - `semi_auto_5m_scan`（每5分钟触发，CRON: `10 */5 * * * *`）
- [x] 添加一个 HTTP Trigger `telegram_webhook`：接收 Telegram Webhook 推送（含安全校验）
- [x] 添加工厂函数 `_create_semi_auto_components()` 和 `_build_semi_auto_engine()`（避免重复代码）
- [x] 更新 `requirements.txt`（新增 `python-telegram-bot>=20.0`、`matplotlib>=3.8.0`）
- [x] 补充 `local.settings.template.json` 中新增的环境变量（`SA_PROFILE`、`CFD_SA_ACCOUNT_SIZE`、`CFD_TELEGRAM_WEBHOOK_SECRET`）
- [x] 修复 `deploy.sh`：将 `utils/` 和 `strategies/` 加入模块复制列表
- [ ] 端到端测试：本地 Azurite + Telegram Bot 联调
