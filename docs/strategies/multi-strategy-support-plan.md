# 多策略支持任务规划

## 背景
当前的实盘策略引擎（`engine.py`）仅支持单一策略（`TrendFilterStateMachine`）。为了提升灵活性和扩展性，需要支持多个策略的动态加载和运行。新策略（`Dual-Direction Trend Trailing System`）已在 TradingView 上验证效果，需要实现为 Python 模块并集成到现有引擎中。

## 目标
1. 支持多个策略的动态加载和运行。
2. 实现新策略 `Dual-Direction Trend Trailing System`。
3. 在本地回测新策略，验证逻辑正确性和性能。
4. 替换或组合现有策略，确保实盘引擎的稳定性。

## 任务分解

### 1. 策略基类设计
- **目标**：定义一个通用的策略基类，所有策略都继承自该基类。
- **文件路径**：`strategies/base_strategy.py`
- **内容**：
  - 定义通用接口，如 `initialize()`、`process_symbol()`。

### 2. 封装现有策略
- **目标**：将现有的 `TrendFilterStateMachine` 逻辑封装为独立的策略类。
- **文件路径**：`strategies/trend_filter_strategy.py`
- **内容**：
  - 实现 `TrendFilterStrategy` 类，继承自 `BaseStrategy`。
  - 将现有逻辑迁移到 `process_symbol()` 方法中。

### 3. 实现新策略
- **目标**：根据文档和 Pine Script 实现 `Dual-Direction Trend Trailing System`。
- **文件路径**：`strategies/dual_direction_trend.py`
- **内容**：
  - 实现 `DualDirectionTrendStrategy` 类，继承自 `BaseStrategy`。
  - 实现入场条件、动态止损、持仓管理等逻辑。

### 4. 引擎重构
- **目标**：修改 `LiveEngine`，支持动态加载和管理多个策略。
- **文件路径**：`live/engine.py`
- **内容**：
  - 增加 `_load_strategies()` 方法，根据配置文件加载策略实例。
  - 修改 `tick()` 方法，调用所有策略的逻辑。

### 5. 配置文件扩展
- **目标**：在配置文件中增加策略相关的配置项。
- **文件路径**：`config/profiles.py`
- **内容**：
  - 增加 `strategies` 字段，定义启用的策略及其参数。

### 6. 本地回测
- **目标**：使用现有回测框架对新策略进行回测。
- **文件路径**：`backtest/run_trend_filter_backtest.py`
- **内容**：
  - 修改回测脚本，支持加载和测试新策略。
  - 输出回测报告，包括收益率、最大回撤、胜率等关键指标。

### 7. 调试与优化
- **目标**：调试新策略，确保逻辑正确性。
- **内容**：
  - 验证新策略在不同市场条件下的表现。
  - 优化性能，减少回测和实盘运行的时间成本。

### 8. 文档更新
- **目标**：更新相关文档，说明新策略的使用方法。
- **文件路径**：`README.md`、`docs/`
- **内容**：
  - 描述多策略支持的设计和使用方法。
  - 提供新策略的配置示例。

## 风险与应对
1. **指标计算问题**：
   - 风险：新策略依赖的指标可能需要扩展现有框架。
   - 应对：检查 `core/indicators.py`，补充缺失的指标实现。

2. **数据问题**：
   - 风险：回测数据可能缺失或不完整。
   - 应对：补充数据或生成模拟数据。

3. **兼容性问题**：
   - 风险：新策略的状态管理可能与现有引擎不兼容。
   - 应对：在集成前充分测试，确保兼容性。

4. **性能问题**：
   - 风险：多策略运行可能增加性能开销。
   - 应对：优化指标计算和状态管理逻辑。

## 交付物
1. 策略基类 `BaseStrategy`。

3. 新策略 `DualDirectionTrendStrategy`。
4. 修改后的引擎 `engine.py`。
5. 更新的配置文件 `profiles.py`。
6. 回测脚本和报告。
7. 更新的文档。
