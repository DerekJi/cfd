## 📋 本次修复内容摘要

### 问题
回测代码直接迁移到实盘，未考虑平台自动止损/止盈：
- FSM 触发平仓信号 → 但持仓已被平台自动平掉
- 重复尝试平仓不存在的持仓 → 大量失败消息
- FSM 状态未同步 → 无法开新仓

### 修复
1. **平仓前验证持仓存在性** - 避免重复平仓
2. **每次 tick 同步持仓状态** - FSM 与实盘保持一致
3. **移除硬止损检查** - 实盘依赖服务端止损
4. **状态修复工具** - 手动修复不一致的状态

### 文件改动
- ✏️ `live/engine.py` - 核心修复
- ✨ `live/fix_state.py` - 新增修复工具

## 🆘 常见问题

### Q1: 如果 `fix_state.py` 报错怎么办？
**A**: 检查环境变量是否设置：
```bash
dcqSetSecrets default
python -c "import os; print('AZURE_STORAGE_CONNECTION' in os.environ)"
```
应输出 `True`。如果是 `False`，重新运行 `dcqSetSecrets default`。

### Q2: 如何确认部署成功？
**A**: 检查 Function App 的部署日志：
```bash
func azure functionapp logstream cfd-trading-live
```
或在 Azure Portal 中查看 Deployment Center。

### Q3: 还是收到平仓失败消息怎么办？
**A**: 
1. 确认已运行 `fix_state.py`
2. 确认已重新部署 Function
3. 检查是否有其他未同步的品种
4. 查看详细日志并联系支持
