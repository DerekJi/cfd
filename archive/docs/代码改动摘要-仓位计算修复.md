## 📝 改动清单

### 1. 修复仓位计算公式（关键修复）✅

**文件**：`backtest/strategies/trend_filter_strategy_v1.py`  
**行号**：L456

#### 修改前（❌ 错误）
```python
# ⚠️ BUG: 对高汇率货币对计算出过大仓位
size = risk_amount * entry_price / (stop_distance * SLIPPAGE_PROTECTION)
```

#### 修改后（✅ 正确）
```python
# ✅ 修复：基于百分比价格变动计算
size = risk_amount / ((stop_distance / entry_price) * SLIPPAGE_PROTECTION)
```

#### 修复说明
- **问题**：旧公式 `size = risk × price / stop` 对高汇率货币对（USDJPY ~150, AUDJPY ~95）会计算出巨大仓位
- **原因**：公式多乘了一个 `entry_price`，导致汇率越高，仓位越大
- **解决**：改为基于百分比变动 `stop_distance / entry_price`，与backtrader盈亏计算一致

#### 影响范围
- ✅ GBPUSD：修复后年化收益15.35%（之前16.67%）- 轻微下降属正常
- ✅ USDJPY：修复后年化收益**35.56%**（之前23.00%）- **大幅提升！**
- ❌ AUDJPY：仍然无法交易（需要专门的交叉盘支持）

### 2. 修复风险金额显示计算 ✅

**文件**：`backtest/strategies/trend_filter_strategy_v1.py`  
**行号**：L312（做多）, L356（做空）

#### 修改前（❌ 错误显示）
```python
# 做多
risk_dollars = size * (entry_price - stop_loss)

# 做空
risk_dollars = abs(size) * (stop_loss - entry_price)
```

#### 修改后（✅ 正确显示）
```python
# 做多
risk_dollars = size * (entry_price - stop_loss) / entry_price

# 做空
risk_dollars = abs(size) * (stop_loss - entry_price) / entry_price
```

#### 修复说明
- **问题**：显示的风险金额与实际盈亏计算不一致（backtrader使用百分比计算）
- **影响**：仅影响debug日志显示，不影响实际交易
- **示例**：
  - 旧显示：`风险金额: $3205.20`（错误）
  - 新显示：`风险金额: $100.00`（正确）

## ⚠️ 已知限制

### AUDJPY交叉货币对不支持
- **原因**：需要USD/JPY实时汇率进行盈亏转换
- **影响**：订单全部因Margin不足被拒（计算保证金错误）
- **解决**：需实现专门的`CrossPairCommissionInfo`类
- **优先级**：Medium（当前GBPUSD+USDJPY已满足需求）

### 其他交叉货币对
以下货币对同样不支持，原因相同：
- EURJPY, GBPJPY（需要USD/JPY汇率）
- EURGBP（需要GBP/USD或EUR/USD汇率）
- AUDNZD,AUDCAD等（需要对应USD交叉汇率）
