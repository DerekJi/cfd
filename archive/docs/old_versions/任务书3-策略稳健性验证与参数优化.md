# 任务书3 - 策略稳健性验证与参数优化

## 📋 任务概述

**目标**：验证趋势过滤策略的稳健性，并优化参数以提升实盘适用性

**背景**：
- 已完成两种策略方案（混合策略 vs 趋势过滤）
- 趋势过滤策略表现更优（盈亏比3.18:1，+0.38%收益）
- 发现ADX=30时性能显著提升（+3.15%收益，回撤仅1.70%）
- 当前仅在GBPUSD上测试，需验证多品种适用性

**当前状态**：
- test/1分支：原始策略（34.3%胜率，504笔交易）
- feature/fade-strategy分支：混合策略（54.2%胜率，-2.66%收益）
- feature/trend-filter分支：趋势过滤策略（24.3%胜率，+0.38%收益）

## 🎯 任务目标

### 主要目标
1. **多品种验证**：测试策略在6个品种上的表现
2. **参数优化**：找到全局最优参数或品种特定参数
3. **稳健性评估**：确认策略是否具备实盘可行性

### 成功标准
- [ ] 至少3个品种表现盈利
- [ ] 收益标准差 < 3%（表现均衡）
- [ ] 所有品种最大回撤 < 5%

## 🎯 关键决策点

### 决策点1：是否需要品种特定参数？（任务3后）

**场景A：表现均衡**
- 收益标准差 < 2%
- 所有品种盈利
- 最佳ADX阈值一致

**决策**：使用统一参数，跳过任务5，直接合并

**场景B：需要优化**
- 部分品种亏损但可调整
- 最佳参数差异大

**决策**：为每个品种设置最优参数，创建配置文件

**场景C：不稳健**
- 多个品种严重亏损
- 更换参数无法改善

**决策**：执行任务5深度优化，或只用于表现好的品种

### 决策点2：是否需要策略优化？（任务4后）

**触发条件**：
- 平均收益 < 2%
- 某些品种回撤 > 5%
- 识别出明显改进点

**决策**：执行任务5，实现V2版本

## ⚠️ 风险和注意事项

### 技术风险
1. **OANDA API限流**：下载可能需要分批进行
2. **数据质量**：某些品种数据可能不完整
3. **计算时间**：多品种测试较耗时

### 业务风险
1. **过拟合**：参数优化可能导致过度拟合历史数据
2. **品种特性**：某些品种可能根本不适合此策略
3. **实盘差异**：回测结果与实盘可能有差距

### 缓解措施
1. 保留验证集（最后1个月数据）
2. 记录所有参数组合，避免选择性报告
3. 设置保守的期望值（实盘打7折）
