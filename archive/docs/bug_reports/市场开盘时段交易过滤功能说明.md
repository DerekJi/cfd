# 市场开盘时段交易过滤功能

## 📋 功能说明

为防止在市场开盘时的高波动/跳空时段开仓造成过大滑点（如USDJPY $614亏损案例），策略新增**市场开盘时段交易禁止**功能。

---

## ⏰ 禁止交易时段

### 1️⃣ 东京市场开盘
**时间窗口**：
- **UTC时间**：23:45 - 00:15（开盘前后15分钟）
- **北京时间**：07:45 - 08:15
- **东京时间**：08:45 - 09:15

**原因**：
- 东京外汇市场09:00开盘
- 日本经济数据通常在开盘前后发布
- USDJPY在此时段波动剧烈，易出现跳空

### 2️⃣ 悉尼市场开盘
**时间窗口**：
- **UTC时间**：20:45 - 21:15（开盘前后15分钟）
- **北京时间**：04:45 - 05:15
- **悉尼时间**：06:45 - 07:15（标准时间）

**原因**：
- 悉尼外汇市场07:00开盘（当地时间）
- 亚太市场第一个开盘，流动性从极低迅速转高
- 易出现价格跳空

## 🛡️ 实施逻辑

### 代码位置
`backtest/strategies/trend_filter_strategy_v1.py`

### 核心函数
```python
def _is_market_open_session(self, dt):
    """
    检查当前时间是否在市场开盘高波动时段（禁止交易）
    
    返回: (is_blocked, session_name)
    - is_blocked: True表示禁止开仓
    - session_name: "东京开盘" 或 "悉尼开盘"
    """
```

### 检查时机
在 `_execute_entry_long()` 和 `_execute_entry_short()` 函数开头：
```python
# 检查市场开盘时段（防止高波动/跳空时段开仓）
dt = self.data.datetime.datetime(0)
is_blocked, session_name = self._is_market_open_session(dt)
if is_blocked:
    self.blocked_by_session += 1
    if self.params.debug:
        print(f"\n⚠️  跳过交易（{session_name}开盘窗口）")
        print(f"   时间: {dt.strftime('%Y-%m-%d %H:%M UTC')}")
        print(f"   原因: {session_name}开盘前后15分钟为高波动时段，禁止开仓\n")
    return  # 放弃开仓
```

## 🎯 典型案例分析

### USDJPY $614亏损案例
**入场时间**：2025-04-16 22:55 UTC  
**平仓时间**：2025-04-17 00:00 UTC（东京开盘时刻）

**现状**：
- ✅ 入场时间 22:55 **不在禁止时段**（允许）
- 🚫 平仓发生在 00:00 **东京开盘时刻**（K线跳空）
- ⚠️ 如果当时是想开仓，**会被阻止**

**分析**：
- 这笔交易的亏损主要因00:00的跳空（high=142.480）
- 虽然无法阻止已持仓时遇到的跳空
- 但可以**防止在00:00±15分钟尝试开新仓**
- 大幅降低开仓时直接遇到跳空的概率

## ⚙️ 时区处理

### UTC时间基准
策略使用**UTC时间（GMT）**作为统一基准：
- 数据源时区：UTC+0
- 所有时间判断基于UTC
- 避免夏令时混乱

### 时区对照表
| 市场 | 当地开盘 | UTC时间 | 北京时间 |
|------|---------|---------|---------|
| 东京 | 09:00 JST | 00:00 | 08:00 |
| 悉尼 | 07:00 AEST | 21:00* | 05:00 |

*注：悉尼夏令时期间为20:00 UTC（当前简化为全年21:00）

## 💡 注意事项

### 1️⃣ 只限制开仓
- ✅ **阻止**：在禁止时段开新仓
- ✅ **允许**：已持仓的止损/止盈操作
- ✅ **允许**：紧急平仓

### 2️⃣ 夏令时简化
当前实现简化处理：
- 悉尼全年按标准时间21:00 UTC
- 实际夏令时期间应为20:00 UTC
- 如需精确，需添加夏令时判断逻辑

### 3️⃣ 统计输出
每次回测结束会显示：
```
市场开盘时段阻止: XX 笔 (悉尼/东京开盘前后15分钟)
```

### 4️⃣ Debug模式
开启 `debug=True` 时，每次阻止交易都会打印：
```
⚠️  跳过交易（东京开盘窗口）
   时间: 2025-04-17 00:05 UTC
   原因: 东京开盘前后15分钟为高波动时段，禁止开仓
```

## 🔧 修改建议

### 调整时间窗口
如需修改禁止时段长度，在 `_is_market_open_session()` 函数中：
```python
# 当前：前后15分钟
if time_minutes >= tokyo_open - 15 and time_minutes <= tokyo_open + 15:

# 改为前后10分钟
if time_minutes >= tokyo_open - 10 and time_minutes <= tokyo_open + 10:

# 改为前后20分钟
if time_minutes >= tokyo_open - 20 and time_minutes <= tokyo_open + 20:
```

### 添加其他市场
可在函数中添加伦敦/纽约开盘：
```python
# 伦敦开盘: 08:00 London = 08:00 UTC (冬令时) 或 07:00 UTC (夏令时)
london_open = 8 * 60
if time_minutes >= london_open - 15 and time_minutes <= london_open + 15:
    return True, "伦敦开盘"

# 纽约开盘: 08:00 NY = 13:00 UTC (冬令时) 或 12:00 UTC (夏令时)
ny_open = 13 * 60
if time_minutes >= ny_open - 15 and time_minutes <= ny_open + 15:
    return True, "纽约开盘"
```

### 完全禁用此功能
如不需要此功能，在入场函数中注释或删除：
```python
# 注释掉这段即可
# is_blocked, session_name = self._is_market_open_session(dt)
# if is_blocked:
#     ...
#     return
```

## 📈 预期效果

### 风险降低
1. **避免直接跳空开仓**：
   - 开盘瞬间价格可能跳空几十点
   - 滑点可达止损距离的3-5倍
   - 直接开仓会面临巨大损失

2. **减少极端滑点**：
   - 开盘前后15分钟流动性不稳定
   - bid-ask spread扩大
   - 成交价偏离预期

3. **保护小仓位策略**：
   - 最小手数(1000 units)在紧密止损下已经风险很高
   - 再叠加开盘跳空，风险成倍增加
   - 此功能是小仓位策略的必要保护

### 收益影响
- **短期**：可能错过少量开盘突破机会（~7%信号）
- **长期**：避免高滑点亏损，整体风险调整后收益提升
- **合规性**：降低单笔巨亏概率，更易通过Blue Guardian评估

## ✅ 总结

### 核心价值
**防止在市场开盘高波动时段开新仓，避免跳空/极端滑点造成的风险失控。**

### 关键特点
- ✅ 仅限制开仓，不影响已有持仓
- ✅ 覆盖悉尼、东京两大亚太开盘时段
- ✅ 可调整时间窗口和市场范围
- ✅ 详细统计和Debug输出
- ✅ 测试显示7.1%交易被阻止，有效降低风险

### 适用场景
- 小仓位策略（最小手数限制严格）
- 紧密止损策略（滑点影响大）
- Blue Guardian等严格风控要求
- USDJPY等亚太时段活跃品种
